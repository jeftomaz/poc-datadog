Bloco 1 · Ícaro
O problema

Software em produção é uma caixa-preta

Manter um sistema não é só corrigir bugs: é entender o que ele faz enquanto roda. Sem visibilidade, a equipe descobre falhas pelo cliente, e o diagnóstico vira tentativa e erro.

Reativo

O problema só aparece quando alguém reclama. O time apaga incêndios em vez de preveni-los.

MTTR alto

Sem dados, achar a causa-raiz leva horas. Cada minuto parado custa confiança e dinheiro.

Adivinhação

"Funciona na minha máquina." Sem telemetria, decisões de manutenção são chutes.

Bloco 1 · Ícaro
Os três pilares

Métricas, Traces e Logs

A observabilidade responde três perguntas diferentes. Juntas, elas reconstroem a história completa de cada requisição.

Métricas
Quanto?

O panorama

Valores numéricos no tempo: requisições/s, latência p95, taxa de erros, uso de heap. Mostram tendências e disparam alertas.

Traces
Onde?

O caminho

Cada requisição vira uma árvore de spans. Revelam onde o tempo é gasto e qual camada falhou.

Logs
Por quê?

O detalhe

Eventos textuais com contexto. Explicam por que algo aconteceu — a evidência fina do diagnóstico.

Bloco 2 · Jeferson
Instrumentação

tracer.js — uma linha que vê tudo

app/src/tracer.js
// carregado ANTES de tudo (1ª linha do server.js)
const tracer = require('dd-trace').init({
  logInjection: true,    // injeta trace_id nos logs
  runtimeMetrics: true,  // heap, event loop, GC
  env: process.env.DD_ENV,
  service: process.env.DD_SERVICE,
  version: process.env.DD_VERSION,
});

module.exports = tracer;
logInjection

Acrescenta trace_id e span_id a cada log. É o que costura log ↔ trace automaticamente.

runtimeMetrics

Coleta métricas do runtime Node — heap, event loop e GC — sem código extra.

Ordem importa

O require('./tracer') é a primeira linha: instrumenta as libs antes do Express carregar.

Bloco 3 · João Lucas
Aprendizados

Problemas encontrados & soluções

● Problema

Logs não apareciam no Datadog

O agente coletava traces e métricas, mas a aba de Logs ficava vazia. A coleta de logs de containers não estava habilitada.

● Solução

Habilitar o Logs Agent

No agente: DD_LOGS_ENABLED=true + DD_LOGS_CONFIG_CONTAINER_COLLECT_ALL=true, e a label com.datadoghq.ad.logs no container da app indicando source: nodejs.

● Problema

Logs chegavam como texto cru / JSON inválido

Sem formato JSON, o Datadog tratava cada linha como string solta — sem nível, sem atributos, sem correlação com o trace.

● Solução

Winston em JSON + logInjection

Padronizar a saída com winston.format.json() e ativar logInjection no dd-trace. Os logs passaram a ser parseados e ligados ao trace_id.

Conclusão

O que a observabilidade entrega à manutenção

Detecção proativa

Monitores avisam antes do usuário. O time age sobre o sintoma, não sobre a reclamação.

Diagnóstico rápido

Trace mostra onde, log mostra por quê. MTTR cai de horas para minutos.

Tendências

Métricas ao longo do tempo guiam decisões de refatoração e capacidade — manutenção baseada em dados.

Observabilidade transforma manutenção reativa em manutenção orientada por evidências.
